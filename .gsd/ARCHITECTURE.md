## Overview

Zonny is a locally-hosted AI development platform with two modes:

1. **Chat Mode** — Multi-agent Q&A with specialist routing (code, docs, memory)
2. **Company Mode** — 6-agent software company that builds entire apps from a single prompt, previews them, and auto-repairs errors

### 2. AutoGen Runtime — `zonny/autogen_runtime.py` (322 lines)

**Architecture:**
```
User task → _select_specialist(keyword routing)
         → Build team: [specialist_agent, assistant_agent]
         → RoundRobinGroupChat with TextMentionTermination("TERMINATE")
```

**Key Functions:**
- `_load_manifests()` — Loads YAML agent configs from `agents/manifests/`

## Data Flow

### Chat Flow
```
User message → POST /mcp
  → autogen_runtime._select_specialist(task)
  → _build_focused_team(task) → [specialist, assistant]
  → RoundRobinGroupChat.run(task)
  → specialist analyses → assistant synthesises + TERMINATE
  → Return {response, conversation, specialist}
```

### Company Flow
```
User prompt → POST /company/stream (SSE)
  → stream_company(prompt, session_id)
  → RoundRobinGroupChat([CEO, Arch, FE, BE, QA, Reviewer])
  → Each agent speaks → _extract_files() → _save_files()
  → Yield {type: "message", agent, content, files_extracted}
  → Final yield {type: "done", ship_report, files}
  → Frontend auto-opens Preview tab
```

### Auto-Repair Flow
```
Preview iframe loads → injected error script catches JS errors
  → postMessage({type: 'preview-errors', errors}) → parent
  → app.js triggerRepair(errors) → POST /company/repair (SSE)
  → stream_repair(session, errors, attempt)
  → Reads existing files from outputs/<session>/
  → Frontend+Backend agents produce fixed code
  → _save_files() → Preview reloads (max 3 attempts)
```

## Conventions

**Naming:** Snake_case for Python, camelCase for JS. Agent names use `{role}_agent` format.
**Structure:** Python modules in `zonny/`, `agents/`, `tools/`, `runtime/`. Frontend is a monolith SPA.
**Config:** YAML manifests for agents, JSON for API keys, env vars for Ollama.
**Streaming:** SSE for all real-time data (no WebSockets except terminal).
**Testing:** None — no test framework or test files present.
