## Overview

**Zonny v0.2.0** is a local-first AI assistant runtime. It exposes three interfaces — an interactive CLI, a FastAPI REST server, and an MCP stdio server — and routes user input through either a fast single-step semantic router or a multi-step planner+executor loop, all backed by a local Ollama LLM and a ChromaDB vector memory store.

## Components

### `server.py` — FastAPI Entry Point
- **Purpose:** Main REST API server. Dual-mode MCP gateway (`POST /mcp`) + legacy chat/upload/agent-management endpoints.
- **Location:** `server.py`

- **Auth:** API key via `Authorization` header, validated against `key.json`

### `orchestrator.py` — Decider-Based Orchestration (Legacy)

- **Flow:** `decide_agents()` → `execute_agents()` → `synthesize_response()`

## Data Flow

### Phase 1 (Simple Input) — Router Mode
```
User Input
  → is_complex_task() = False
  → semantic_router.route()  [LLM call → JSON intent]
  → dispatcher.dispatch()    [intent → tool execution]
  → Return response
```

### Phase 2 (Complex Input) — Planner Mode
```
User Input
  → is_complex_task() = True
  → planner.plan()           [LLM call → multi-step plan]
  → executor.execute_plan()  [step-by-step tool calls]
  → reflector.reflect()      [quality evaluation]
  → get_final_result()
  → Return response + execution metadata
```

### Legacy Chat Flow (`/v1/chat/completions`)
```
User Message
  → orchestrator.decide_agents()  [LLM → which agents to invoke]
  → orchestrator.execute_agents() [document / memory / code agents]
  → orchestrator.synthesize_response() [final LLM synthesis]
  → store_message() to ChromaDB
  → Return OpenAI-compatible response
```

### Document Ingest (`/v1/upload`)
```
File Upload (PDF or text)
  → PdfReader (if PDF) → extract text
  → chunk_text() [500 char chunks, 100 overlap]
  → embedder.encode() [all-MiniLM-L6-v2]
  → ChromaDB collection.add()
  → Return document_id
```
